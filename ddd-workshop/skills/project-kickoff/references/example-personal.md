# 프로젝트 정체성: 개인 독서 기록 도구

## 개요
- **한 줄 정의**: 읽은 책과 인상 깊은 문장을 모으고, 주제별로 다시 꺼내 볼 수 있는 개인용 도구.
- **맥락**: 개인 도구
- **해결하려는 문제 (Why)**: 노션/굿리즈/메모 앱에 흩어진 독서 메모를 한 곳에서 관리하고, 5년 뒤에도 "내가 이 주제에 대해 어떤 책을 읽었지?"를 빠르게 답할 수 있게 한다.
- **핵심 가치**: 장기 유지 가능한 개인 지식 자산화.

## 사용자·액터
- 주요 액터: 본인

## 범위
- 반드시 포함:
  - 책 등록 (ISBN or 제목 수기 입력)
  - 문장 발췌 + 태그
  - 주제·태그별 검색
  - 로컬 파일 백업 (Markdown/JSON)
- 명시적 제외 (영영 안 함):
  - 다른 사용자와의 공유
  - SNS 연동, 공개 프로필
- 나중으로 미룸:
  - OCR로 종이책 문장 스캔 (v2)
  - 모바일 앱 (웹 PWA로 충분한지 먼저 검증)

## 제약 조건
- 기술: 본인에게 익숙한 Next.js + SQLite. 호스팅은 홈서버 or Vercel.
- 일정·인력: 혼자. 유지보수 부담을 줄여야 함.
- 장기 운영: 5년 이상 데이터 유실 없이 써야 함 → export·백업 필수.

## 미검증 가정 / 리스크
- ⚠️ ISBN 메타데이터 API(구글북스 vs 알라딘)의 무료 한도·정책 변경 가능성
- ⚠️ 가장 큰 리스크는 "귀찮아서 안 쓰게 되는 것". 입력 마찰을 최소화해야 함 → UX가 핵심
