# 프로젝트 정체성: 제조사향 품질관리(QMS) SaaS

## 개요
- **한 줄 정의**: 중소·중견 제조사가 공정 검사·부적합 관리·시정조치(CAPA)를 온라인으로 처리하는 B2B SaaS.
- **맥락**: B2B
- **해결하려는 문제 (Why)**: 대부분의 중소 제조사가 엑셀·종이로 품질 기록을 관리하고 있어 ISO 9001 감사 대응에 매번 과도한 비용이 든다.
- **핵심 가치**: ISO 9001 인증 유지 공수를 1/3로 단축.

## 사용자·액터
- 주요 액터 (고객사 내부):
  - 현장 검사원 (모바일에서 검사 기록 입력)
  - 품질 관리자 (부적합 승인, CAPA 발행)
  - 경영진 (대시보드 조회)
- 감사자 (외부): 고객사 임시 접근 계정으로 조회

## 범위
- 반드시 포함:
  - 검사 기준서 등록·버전 관리
  - 모바일에서 검사 기록 입력(오프라인 지원)
  - 부적합 발생 → CAPA 워크플로
  - ISO 9001 요구 양식에 맞는 리포트 생성
  - 감사자 읽기 전용 접근
- 명시적 제외 (영영 안 함):
  - MES·ERP의 실시간 공정 제어 영역 (연동만)
- 나중으로 미룸:
  - 통계적 공정 관리(SPC) 차트 자동화 — v1.5

## 제약 조건
- 기술: AWS 멀티테넌트 SaaS, 고객사별 데이터 격리 필수
- 일정·인력: 개발 6인, 파일럿 고객 3사, 6개월 내 GA 목표
- **계약 단위**: 고객사당 월 구독 + 좌석 수 기반 요금제
- **멀티테넌시**: 고객사별 데이터 완전 격리 (단일 DB 스키마 내 tenant_id 분리 vs 고객사별 DB 분리 — 결정 필요)
- **SLA**: 영업시간 99.9%, 야간 배치창 24:00~02:00
- **보안**: ISO 27001 준비, 고객사 요청 시 데이터 반환·파기 증명
- **커스터마이징**: 고객사별 검사 기준서 템플릿 차이 허용, 코드 수정 없이 설정으로 대응

## 이해관계자
- 고객사 의사결정자: 품질 관리자, IT 담당자, 구매 담당자
- 최종 사용자: 현장 검사원 (IT 리터러시 낮음)
- 감사 기관: 외부 인증기관 (KS 인증, TÜV 등)

## 미검증 가정 / 리스크
- ⚠️ 중소 제조사의 모바일 현장 환경(네트워크 음영, 저사양 기기)에 대한 구체 데이터 없음 → 파일럿 고객 방문 필요
- ⚠️ ISO 9001:2015 조항 중 어디까지를 "반드시 디지털 제공"으로 볼지 컨설턴트 자문 필요
- ⚠️ 경쟁 제품(Qualio, Greenlight Guru 등)이 이미 국내에 진입해 있는지 조사 필요 → domain-research
