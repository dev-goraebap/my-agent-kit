# 프로젝트 정체성: 영업팀 견적 관리 시스템

## 개요
- **한 줄 정의**: 영업팀이 고객별 견적서를 만들고, 승인·발송·이력 관리를 하나의 워크플로로 처리하는 사내 도구.
- **맥락**: 사내 도구/솔루션
- **해결하려는 문제 (Why)**: 현재 견적은 Excel + 메일로 오가며, 승인 이력이 산재되어 감사 대응이 어렵다. 같은 고객에게 담당자별로 다른 단가가 나가는 사고도 종종 발생한다.
- **핵심 가치**: 견적 단가 일관성 보장 + 감사 추적성.

## 사용자·액터
- 주요 액터: 영업 담당자, 영업팀장(승인자), 재무팀(감사), 고객(외부, 견적서 수령)
- 현재 관리 대상 사용자 수: 약 40명 (영업 30, 승인 5, 감사 5)

## 범위
- 반드시 포함:
  - 고객·품목 마스터 관리
  - 견적서 작성·승인 워크플로
  - PDF 출력 + 이메일 발송
  - 모든 상태 변경 이력 보관 (감사 요건)
- 명시적 제외 (영영 안 함):
  - 실제 수주·매출 기록 (이미 SAP가 담당)
  - 재고 관리 (별도 WMS 존재)
- 나중으로 미룸:
  - 고객 포털 (고객이 견적을 직접 확인) — Phase 2

## 제약 조건
- 기술: 사내 표준 스택 (Spring Boot + PostgreSQL + 사내 SSO)
- 일정·인력: 개발 2인, 목표 3개월 내 파일럿
- 연동 시스템:
  - SAP (고객·품목 마스터 동기화, 야간 배치)
  - 사내 SSO (Keycloak)
  - 사내 메일 게이트웨이 (발송 로그 필수)
- 규제·감사:
  - 내부 감사 정책상 견적 발송 이력 **7년 보관**
  - 단가 변경은 승인자 2인 이상 서명 필요

## As-Is 프로세스 (현재 방식)
1. 영업 담당자가 Excel 템플릿으로 견적 작성
2. 팀장에게 메일로 송부 → 메일 답장으로 승인
3. PDF로 변환 후 고객에게 메일 송부
4. 사본을 공유 폴더 연도별 디렉토리에 수기 저장

## 이해관계자
- 사용자 부서: 영업팀 (요구사항 주도)
- 승인자: 영업팀장
- 감사 요구사항 제공: 재무팀, 내부감사팀
- 기간계 연동: SAP 운영팀

## 미검증 가정 / 리스크
- ⚠️ SAP 고객 마스터 필드 구조 미확인 → 연동 설계 전 SAP 운영팀과 미팅 필요
- ⚠️ "단가 변경 승인 2인 서명" 정책이 전자서명으로 인정되는지 법무팀 확인 필요
- ⚠️ 현재 Excel에 특수한 할인 로직이 있다면(사용자가 언급 안 함) 놓칠 수 있음 → domain-research + requirement-clarifier에서 반드시 끄집어낼 것
