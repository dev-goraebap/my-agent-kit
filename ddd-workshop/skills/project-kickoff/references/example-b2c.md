# 프로젝트 정체성: 지역 기반 중고거래 플랫폼

## 개요
- **한 줄 정의**: 동·읍·면 단위 지역에서 개인이 중고물품을 안전하게 거래할 수 있는 B2C 플랫폼.
- **맥락**: B2C 제품
- **해결하려는 문제 (Why)**: 기존 중고거래는 지역 기반이지만 사기·분쟁·매너 이슈로 신뢰가 낮고, 탈중앙 오픈마켓은 배송·수수료 부담이 크다. **지역 기반 신뢰**를 재료로 마찰을 낮춘다.
- **핵심 가치**: 같은 동네 사람들끼리의 낮은 거래 마찰 + 안전장치.

## 사용자·액터
- 주요 액터:
  - 구매자 (일반 회원)
  - 판매자 (일반 회원, 역할이 구매자와 겹침)
  - 분쟁 조정 운영자 (내부)
  - 고객센터 상담원 (내부)
- 타깃 사용자:
  - 1차: 20~40대 동네 거주 회원, 주 1회 이상 중고거래 이용 경험
  - 2차: 아이 키우는 가정(유아용품·장난감 회전 수요 높음)

## 범위
- 반드시 포함:
  - 위치 인증(동 단위)
  - 상품 등록·검색·채팅
  - 매너 온도/후기
  - 신고·분쟁 조정
  - 금지 품목 필터링
- 명시적 제외 (영영 안 함):
  - 배송 대행 (지역 직거래에 집중)
  - 개인 간 에스크로 금융 서비스 (v1에서는 규제·비용 리스크)
- 나중으로 미룸:
  - 동네 커뮤니티 글/모임 (v2)
  - 소상공인 프로 계정 (v1.5)

## 제약 조건
- 기술: React Native + NestJS + PostgreSQL + PostGIS
- 일정·인력: 개발 5인, 6개월 내 수도권 5개 동 베타
- **타깃 플랫폼**: iOS, Android (웹은 조회만)

## 차별점 (경쟁 대비)
- 당근마켓 대비: 매너 온도보다 **구체적 거래 이력 신뢰지표** 제공
- 번개장터 대비: **지역 직거래 전용**, 택배 거래 일체 배제
- 중고나라 대비: **앱 네이티브**, 사기 탐지 강화

## 성공 지표 (Phase 1 베타)
- 동당 주간 활성 사용자 300명 이상
- 등록 → 첫 거래 전환율 20% 이상
- 신고 건수 대비 분쟁 조정 완료 시간 중앙값 24시간 이내
- 사기 피해 발생률 < 0.5%

## 미검증 가정 / 리스크
- ⚠️ 한국 중고거래 시장의 금지 품목 규정 (담배·주류·의약품 등 법률 경계) → domain-research 필수
- ⚠️ 개인 간 거래의 세무 처리 기준 (일정 금액/횟수 이상 시 통신판매업 신고 의무?) → 리서치 필요
- ⚠️ 위치 정보 처리의 개인정보 규제(개보법 제15조 관련) 해석
- ⚠️ "분쟁 조정"을 내부 운영이 얼마나 감당할 수 있는지 인력 산정 필요
- ⚠️ 최근 2년간 중고거래 사기 유형 변화 → 리서치로 점검

## 다음 단계
B2C 맥락은 **규제·경쟁·사용자 행동**이 도메인 설계 전반에 영향을 준다. `domain-research`에서 다음을 확보:
- 중고거래 관련 법규 (금지 품목, 통신판매업, 세무)
- 경쟁 서비스 3종의 도메인 관행(신뢰 지표, 분쟁 처리)
- 최근 사기 유형 3~5종 및 업계 대응 방식
- 위치 정보·개인정보 처리 기준
