# 도메인 브리프: 제조사 QMS (품질관리 시스템) 도메인

## 에이전트 자가 진단

### ✅ 일반 상식 수준에서 아는 것
- "품질관리"라는 용어의 일반 개념
- ISO 9001이 품질경영시스템 국제 표준이라는 사실
- "부적합(Nonconformance)", "CAPA(시정및예방조치)"라는 용어가 존재한다는 수준

### ⚠️ 얕게 아는 것
- ISO 9001:2015의 구조(10개 조항)
  - 왜 얕은가: 조항 번호·명칭만 알고 실제 기업 운용 방식은 모름
- CAPA 프로세스의 일반 흐름
  - 왜 얕은가: 개념적 흐름만 알고 업계별 구현 차이 모름

### ❌ 모르는 것
- 한국 제조업 인증 생태계 (KS, Q마크, TÜV, DNV 등 인증기관의 실제 영향력)
- 중소 제조사의 현장 IT 수준 (모바일 음영, 기기 스펙)
- 경쟁 QMS SaaS (Qualio, Greenlight Guru, MasterControl 등)의 가격·국내 진입 현황
- 외부 감사 시 실제로 요구되는 전자 기록 수준
- 최근 1~2년 ISO 9001 개정 움직임 (ISO 9001:2026 논의 등)

## 업계 일반 지식
- ISO 9001:2015는 PDCA(Plan-Do-Check-Act) 순환을 핵심 구조로 한다.
  - 출처: [ISO 9001:2015 개요](https://www.iso.org/iso-9001-quality-management.html) — 2026-04-21 조회
- 부적합 관리는 (1) 발견 (2) 격리 (3) 원인 분석 (4) 시정조치 (5) 예방조치 (6) 유효성 검증 단계로 구성되는 것이 일반적.
  - 출처: [ASQ CAPA 개요](https://asq.org/quality-resources/capa) — 2026-04-21 조회

## 관련 법규 및 규제
- **산업표준화법**: KS 인증 관련 법적 근거.
  - 출처: [국가법령정보센터](https://www.law.go.kr/) — 2026-04-21 조회
  - ⚠️ 특정 제품군별(식품, 의료기기) 추가 규제가 병렬로 존재 — 고객사 제품군 확인 필수.
- **전자문서 및 전자거래 기본법**: 검사 기록의 전자적 보관 효력 근거.
  - 출처: [국가법령정보센터](https://www.law.go.kr/) — 2026-04-21 조회
  - ⚠️ "원본성·무결성" 요건(타임스탬프, 변경 이력) 기술적 충족 방식 세부 확인 필요.
- ⚠️ 실제 감사 기준·해석은 **인증기관 심사원·법무 자문 필수**.

## 업계 표준 프로세스 (Quality Operations)
1. 검사 기준서(Inspection Specification) 승인·배포
2. 현장 검사 실시 → 적합/부적합 판정
3. 부적합 발생 시 격리·NCR(Non-Conformance Report) 발행
4. 근본원인분석(Root Cause Analysis, 5Why·어골도)
5. CAPA 수립·승인·실행
6. 유효성 검증 후 종결
7. 내부 감사(Internal Audit) → 경영검토(Management Review)
- 출처: [ISO 9001 표준 개요 공개 자료 + ASQ]

## 주요 플레이어 및 관행
- **Qualio**: 클라우드 QMS, 생명과학 중심. 전자서명·감사추적 강점.
  - 출처: [Qualio 공식 사이트](https://www.qualio.com/) — 2026-04-21 조회
  - ⚠️ 국내 진입 규모 불명, 가격 비공개.
- **Greenlight Guru**: 의료기기 특화 QMS.
  - ⚠️ 제조업 일반에 그대로 맞진 않음.
- **MasterControl**: 제약·의료기기 중심의 엔터프라이즈 QMS.
- **국내 경쟁**: ⚠️ 확인 필요. 많은 중소 제조사가 아직 엑셀·사내 전사시스템 커스텀 사용 추정.

## 자주 발생하는 엣지 케이스
- ⚠️ **업계별 상세 케이스는 확인 부족**. 파일럿 고객 3사의 실제 부적합 유형 수집 필요.
- 일반적으로:
  - 외부 공급업체의 부적합이 발견되는 경우 (공급업체 CAPA 요구)
  - 이미 출하된 제품의 부적합 (리콜·고객 통지)
  - 측정 기기 검·교정 주기 초과 (해당 기간 검사 결과 전수 재검토)

## ⚠️ 여전히 불확실한 것
- 한국 중소 제조사가 ISO 9001 외에 **실제로 운용하는** 품질 표준 조합 (KS, 고객사 요구 표준 등).
- 현장 작업자의 실제 IT 리터러시·모바일 사용 패턴 (추측 금지, 파일럿으로 확인).
- 감사 시 "전자 기록"으로 인정되는 포맷·메타데이터 요구 수준.
- 경쟁 SaaS의 국내 실질 점유.

## ⚠️ 사용자에게 확인 필요한 것 (조직·프로젝트 고유 맥락)
- 타깃 제품군이 식품·의료기기·자동차 등 **추가 규제 업종**인가?
- 파일럿 고객 3사의 현재 부적합 관리 방식 (엑셀? 자체 DB? 종이?).
- 고객사별 커스터마이징 범위 (템플릿만? 승인 라인 구조까지?).
- 감사 대응 실무 파트너(컨설턴트·인증기관)가 있는가?

## 후속 스킬을 위한 힌트

### requirement-clarifier가 물어야 할 업계 특화 질문
- 검사 기준서의 버전 관리는 누가·언제·어떤 조건에서 개정되는가?
- 부적합 발생 → NCR 발행 → 격리 플로우에서 현장 작업자와 관리자의 역할 분리?
- CAPA 유효성 검증의 주기와 판정자는?
- 측정기 검·교정 기록은 이 시스템 범위인가, 별도 시스템인가?
- 공급업체 부적합 발생 시 공급업체와의 커뮤니케이션 방식(이메일? 포털?)
- 리콜 의사결정 권한·프로세스?

### ubiquitous-language-extractor가 주의할 용어
- **부적합 / 결함 / 불량 / 비적합** — 업계 표준은 "부적합(Nonconformance)"이지만 현장에서는 "불량"이 더 자주 쓰임. BC 분리 힌트 가능성.
- **CAPA / 시정조치 / 예방조치** — 영문·국문·약어 혼용.
- **검사(Inspection) / 시험(Test) / 검증(Verification)** — 영어 원어에서 의미 구분되나 한국어로는 혼용되기 쉬움.

### event-flow-mapper가 누락하기 쉬운 이벤트
- NCR **격리(Quarantine)** 이벤트 — 부적합 발견과 CAPA 사이에 반드시 들어가는 단계인데 서비스 중심 모델링에서 자주 빠짐.
- 검사 기준서 **개정 배포됨** 이벤트 — 버전 관리의 핵심.
- 측정기 **검교정 만료임박** / **만료됨** 이벤트.
- 감사 **관찰사항(Observation)** 발행 이벤트 — 부적합과 구분.

### bounded-context-identifier가 참고할 구분선
- [검사 기준서(Specification Management)] [검사 실행(Inspection)] [부적합·CAPA] [감사(Audit)] 가 자연스럽게 분리될 가능성 높음. 각 영역의 "용어"가 다름을 확인할 것.

### strategic-classifier가 참고할 경쟁 차별점
- Qualio·Greenlight Guru는 **생명과학·의료기기** 특화. 일반 중소 제조업은 공백.
- Core 후보: "중소 제조업 현장에 맞춘 **모바일 부적합 입력 + CAPA 단순화**" (project-kickoff의 Why와 정합).
- Supporting 후보: 검사 기준서 관리(표준 패턴).
- Generic 후보: 사용자·권한, 감사 로그 → 외부 솔루션 대체 가능.
